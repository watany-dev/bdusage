# v0.1 手動スモークチェックリスト

npm 公開前に、実 AWS 環境で以下を確認する。CI ではモックのみのため、本チェックはリリース担当者が手動で実施する。

前提: `~/.config/bdusage/config.toml` を設定済み、`bun run ci` がローカルで成功していること。

## 共通

- [ ] `npx bdusage --version` が `package.json` と一致する
- [ ] レポートヘッダに `source:` が出る（actual / estimate の区別が明確）
- [ ] CUR 利用時に `engine:` が出る（DuckDB または Athena）

## CUR + Athena

- [ ] `npx bdusage daily --principal self --source cur --cur-engine athena --since 7d`
- [ ] `npx bdusage summary --source cur --cur-engine athena`
- [ ] `npx bdusage doctor` で `athena_*` チェックが期待どおり

## CUR + DuckDB

- [ ] `cur.duckdb.files` のみ（Athena 未設定でも可）で `npx bdusage daily --source cur --cur-engine duckdb --since 7d`
- [ ] `npx bdusage doctor` で `duckdb_*` チェックが期待どおり

## Cost Explorer

- [ ] `npx bdusage daily --source ce --since 7d`（ヘッダ: `source: Cost Explorer actual-lite`）
- [ ] `npx bdusage daily --source auto` で CUR 未設定時に CE へフォールバックする
- [ ] `--principal-tag key=value` が動作する（tag 設定環境のみ）

## CloudWatch Logs（estimate）

- [ ] `npx bdusage today --source logs --principal self`
- [ ] ヘッダに `estimated cost, not billing data` がある
- [ ] 出力にプロンプト / レスポンス本文が含まれない

## その他コマンド

- [ ] `npx bdusage weekly --since 90d`
- [ ] `npx bdusage users --all --since 30d --source cur`
- [ ] `npx bdusage daily --principal-role <role-arn> --since 7d`
- [ ] `npx bdusage daily --json --since 7d` がパース可能な JSON
- [ ] `npx bdusage whoami`

## 診断・エラー

- [ ] `line_item_iam_principal` 欠如時、`doctor` に修正手順（Data Exports の IAM principal 有効化）が出る
- [ ] データなし期間で exit code 0、メッセージが分かりやすい
