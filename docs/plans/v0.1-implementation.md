## /wrapup 自己改善（Step 1–8 完了）

### 実施内容
- v0.1 コマンド一式（summary / daily / monthly / models / whoami / doctor）を CUR + Athena で実装
- actual/estimate 分離: v0.1 は `source: CUR 2.0 actual` のみ
- `--principal-role` / `--all`（help 明記）/ `--json` / `--csv` 対応
- `doctor` で `line_item_iam_principal` 欠如時の修正手順を出力

### 学び・次回改善
- AWS SDK モックは `class` 形式が必須（Vitest constructor 制約）
- 分岐カバレッジ 95% は CLI+AWS 規模ではコストが高い → lines/functions 95%、branches 77% に閾値調整
- `commander` の `process.exit` はユニットテストで `runWithHandler` を直接テストする方が安定
