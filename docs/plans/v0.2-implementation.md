## Step 1 設計

- `V02_SOURCES`: `cur` | `ce` | `auto`
- `auto`: CUR を試行 → 失敗時 CE → 両方失敗時 doctor 案内（SPEC §6.4）
- `--principal-tag key=value`: CE の cost allocation tag フィルタのみ（`--source ce` または auto→ce 時）
- CE では IAM principal ARN フィルタ不可（`self`/`arn`/`role` は明示エラー）
- レポートヘッダ: `source: Cost Explorer actual-lite`（actual-lite は actual と混在しない）
- 集計: `USAGE_TYPE` グループ → 既存 `mapRaw*Rows` を再利用

## /wrapup 自己改善（Step 1–6 完了）

### 実施内容
- `--source ce` / `--source auto`（cur → ce フォールバック）を実装
- `--principal-tag key=value` で Cost Explorer の cost allocation tag フィルタ
- `USAGE_TYPE` グループ集計を CUR と同じ aggregate パイプラインで表示

### 学び・次回改善
- CE は IAM principal ARN をフィルタできないため、CUR/CE で principal 解決を分岐させた
- `doctor` の早期 return が CE 診断をスキップしていた — 常に `appendCeChecks` するよう構造変更
- カバレッジ維持のため `LiveCostExplorerClient` と `createBillingSource` のモックテストを追加
